## Architecture changes (high-level)

### Auto-recall + auto-remember (the memory continuity system)

* **`core/agent._build_auto_recall_note(user_message)`** — extracts 3-5
  keywords from user message, runs `memory.search_facts` per keyword,
  merges top 3 dedup'd hits into the leading system prompt. Filters out
  noise tags (`superseded`, `intervention,auto`, `prune,auto`,
  `credential_scrub`, `blocked_url`). Cap: 3 facts × 200 chars (~1500
  total). Skips `goal:` / `heartbeat:` convs (they have their own
  context).
* **`core/agent._autoremember_findings(reply, conv_id)`** — 9 regex
  patterns scan final assistant reply for finding-shaped statements
  ("Found at /path/", "Located in X", "It's at /Users/...", any
  sentence containing an absolute path) and persist them as facts
  tagged `auto_finding,extracted_from_reply`. Caps at 4/reply. NOW
  ALSO fires on goal-tick replies (was excluded in earlier version).
* **7 seeded facts in `long_term_facts`** auto-surface for queries
  about coding/training/URLs — pre-populated knowledge of the
  `/Users/home/Desktop/Charles URLS/` directory structure +
  behavioural directives ("read source file directly, don't grep with
  guessed keywords"). See `tags='reference,file_location,…'`.

### Tool-call guards (`core/tool_guards.py`)

* **URL block-list (per-conv, persisted)** — when `browse_url` returns
  `[BLOCKED reason=…]` (or matches legacy phrase patterns: "Access
  denied", "Just a moment", "Page Not Found", etc.), the URL is
  poisoned for that conv. Future calls short-circuit with
  `[error] you already tried this URL earlier`. **Persisted to
  `long_term_facts` tagged `blocked_url,blocked_url:<reason>,conv:<id>`**
  so a process restart doesn't lose the dead-URL knowledge.
  `respond_started` rehydrates the in-memory dict from these facts.
* **Same-call dedup with escalation** — `_in_flight: dict[sig, count]`
  tracks (tool_name + json args) within ONE respond chain. 1st repeat
  → "you already called…", attempt #3+ → "STOP. You have now called…
  pick a DIFFERENT tool or call complete_goal." Catches
  verywellmind×4 retry pattern from 2026-05-09 night.
* **read_file content cache (per chain)** — re-reading the same path
  in one respond returns `[cached read_file]` with the SHA-256 prefix
  instead of re-dumping the file.
* **sqlite3-on-memory.db redirect** — `_OWN_DB_PATTERN` regex catches
  `exec_shell` commands containing `sqlite3 … memory.db` and returns
  a redirect message to use `recall()` / `search_facts()` instead.
  Tested live and works.
* **Search-loop nudge** — 4+ exec_shell with grep/find/rg/ag/locate in
  one chain → 5th gets `"[error] you've run N grep/find commands…
  STOP iterating searches. read_file directly."`
* **Fuzzy-recall nudge** — 4+ recall() calls returning <100 chars in
  one chain → 5th gets `"[error] you've made N recall() calls… your
  tag schema is probably wrong. Try recall('<broad>') or
  search_facts() or list_goals()."`

### Watchdog (`scripts/behavior_watchdog.py`)

* **Boss Hog voice** — all `friendly` strings rewritten in foreman
  blue-collar voice (commit ee28ed5). Tags John as "boss" on bigger
  reports. Voice doctrine documented at top of this handover.
* **Hallucination terms tightened** — dropped "ford" / "luxury car"
  / "company about page" from `HALLUCINATION_TERMS`. Caused
  false-positive on legitimate Ford Foundation references in
  psychology research. Kept Tesla/Bugatti/Rivian/larsonjuis (specific
  fabrications, no legitimate use case).
* **Tool-error-storm filter** — excludes guard-issued errors from
  the storm count. Phrases: "you already called", "you already tried
  this URL", "your own memory database", "STOP. You have now called",
  "Re-emit your tool_call", "you've run N grep/find", "you've made",
  "Your tag schema assumption", "[cached read_file]".
* **10-min cooldown on borderline loop-skip warnings** — when
  detector flags but strict trim disagrees, log only once per conv
  per 10 min. Uses in-memory `_LOOP_SKIP_COOLDOWNS` dict (NOT the
  state file — `tick()` races intervenors on state saves and would
  overwrite). Process-local; on restart logs once per conv then
  goes quiet.
* **Auto-prune** — 200 conv rows/conv_id, 7d daily_log retention,
  cap 10 notes/goal (wipe on cancelled/done), 14d self_modify_backups,
  1h audio tmp, 50K hard ceiling on conversations table.

### Async API + Live ticker (the UX win)

* **`POST /api/command/message`** — returns 200 within ~250ms with
  a placeholder reply string; runs `agent.respond` in a background
  asyncio task. UI's existing 2-sec polling picks up turns + the
  mutating progress ticker as Charles works. No more 60-sec UI
  timeouts. Commit be2f508.
* **Single-row updating ticker** — `memory.insert_progress() →
  update_progress() → delete_progress()`. agent.respond inserts ONE
  `role='progress'` row at chain start (`*workin' on it…*`), updates
  its content as each tool fires (`*Pullin' up wikipedia.org…* →
  *Pullin' up wikipedia.org → 16,044 chars*`), deletes it once the
  final reply lands. UI sees one mutating italic line, conv stays
  clean post-reply.
* **Mac UI auto-poll** (`ConversationView.swift`): `.task(id:
  selectedConvId)` runs an async loop calling `loadTurns(id)` every
  2 sec while a conv is selected. Replies + ticker now appear
  without a manual refresh.

* **`warroom/server._normalize_user_conv_id`** is now an

  `heartbeat:` / `sunday_test_` routes to John's primary

* **Mass migration** — 252 turns from 31 leftover test/UI conv ids

  `/api/state/conversations/{conv_id}`.

### Forced behavioural recovery paths

* **Forced summary on MAX_TOOL_ROUNDS** — when for-loop exhausts
  without breaking, run one more `complete()` call with
  `tools=None` and a "stop calling tools, write a 200-word summary"
  prompt. Guarantees the user gets a synthesis instead of a
  mid-tool-chain text fragment.
* **Narration-stall recovery on chain-end** — if final reply
  starts with "now let me", "let me save", "I'll continue", etc.
  AND is short AND lacks code/path content, force one more
  `complete()` call with strict "no more 'let me' phrases — write
  what you actually did or call write_file" prompt.
* **System prompt MANDATORY** — Charles's grounding text now
  explicitly mandates `set_goal` as the FIRST tool call on
  long-running directives (trigger phrases listed). Without a
  goal, work dies when the chain ends. Commit c06c006.

   `async_test`. Cosmetic only.

   tick (oldest-ripe). If multiple goals are active, advance
   cadence per goal stretches. This is by design — single-track
   focus. If John ever creates many concurrent goals expect each

   MAX_TOOL_ROUNDS as user-message respond. With ~25K-100K prompt
   tokens (after several scraped pages accumulate), MLX may take
   2-9 minutes per chain. If the chain is in flight when you

   KeepAlive respawns cleanly. If you see `-9` repeatedly in

## Forensic record — failure modes addressed

For posterity / pattern recognition for future debugging:

| Failure mode | Symptom | Fix |
|---|---|---|
| Hallucinated completion | "10 URLs scraped!" with zero tool calls | auto-remember + verify-before-claim doctrine fact + watchdog hallucination guard |
| 4000-token reasoning leak | completion_tokens=4000, content=empty (Qwen "thinking" tokens flushed to msg.reasoning) | round-0 max_tokens=1500 cap; empty-content fallback marker |
| UI 60-sec timeout | sync API blocks past UI's HTTP timeout | async API |
| Repeated dead-URL retries (×48 ResearchGate) | no enforcement of "already failed" memory | URL block-list, persisted to facts |
| sqlite3-on-memory.db (×56 in one goal) | wrong tool for self-querying | dispatcher redirects to recall() |
| read_file × 55× same path | no caching | per-chain content-hash cache |
| grep iteration loops (17 same-shape commands) | no fuzzy dedup | search-loop nudge after 4 |
| recall iteration loops (url:1, url:11, url:12…) | wrong tag schema, no enforcement | fuzzy-recall nudge after 4 short results |
| Path-typo loops | "Initial training data" (no .txt), 45 turns later same typo | file-not-found-with-suggestions in `tools/filesystem.py` |
| Same-call dedup dodged by minor arg changes | `grep /path1`, `grep /path2`, etc. | escalating STOP message after 2nd block |
| Channel fragmentation | UI sent to leftover stress test conv, broke memory | unconditional reroute + 252-turn migration |
| Watchdog false-positive (Ford Foundation) | "ford" too generic | tightened HALLUCINATION_TERMS |
| Watchdog false-positive (storm warnings on guards) | guard errors counted toward storm | _GUARD_ERROR_PHRASES filter |
| Watchdog log spam (every 30s) | borderline loops fire repeatedly | 10-min cooldown |
| Cooldown state lost (race) | tick() overwrote intervenor's state save | in-memory dict |
| Mid-thought chain ends | `tool_calls=0` breaks loop, narration text logged as final | narration-stall recovery |
| MAX_TOOL_ROUNDS exhaust without summary | "(this tick used the full tool budget)" | forced summary call |
