The 62 unused tools include core-feature surfaces: `vibe_check`, `run_watchdog`, `watchdog_history`, `send_imessage`, `notify_john`, `recent_imessages`, `john_says`, `john_pref_add`, `topic_list/tree/recall_topic`, `skill_register/log/list/get`, `consolidate_memory`, `reflect_now`, `triage_inbox`, `track_open_request`, `analyze_sentiment`, `triangulate`, `self_modify/self_patch`, `start_persistent_task`, `schedule_task`, `add_task`, `current_time`, `get_weather`, `search_web`, `solve_recaptcha`, plus all calendar+email tools.

**Caveats** (don't just delete):
- `tool_calls_json` is only populated in `CHARLES_LOG` (88 rows), not `JOHN_CHARLES` by design. So if Charles called these in the relational channel, we wouldn't see them here.
- Many (`notify_john`, `send_imessage`, `consolidate_memory`, `reflect_now`, `get_weather`) are invoked by **scheduler/heartbeat code paths**, not via the LLM tool-call mechanism — they still need to exist as callable Python.
- Schedule references confirm: task #9 = weather, #11 = consolidate_memory, #14 = reflect_now.

**Real proposal:** keep all tool **functions**, but stop loading every tool's schema into every prompt. Three tiers:
1. **Always loaded** (~10 tools): the actually-used set + critical safety (`reset_my_conversation`, etc.)
2. **On-demand** (`+toolset:email`, `+toolset:calendar`, `+toolset:topics`): Charles loads these when relevant
3. **System-only** (never in prompt): tools called by scheduler/heartbeat only — `consolidate_memory`, `reflect_now`, etc.

Estimated prompt savings: ~3-5k tokens per chain. With ~100 chains/day on busy days = 300-500k tokens/day saved. Probably 30-50% latency drop on tool-routing rounds.

**Risk:** medium-high. Needs a tool-routing layer in `core/prompts.py` and care that on-demand loading doesn't introduce a new round trip. Don't ship without measuring vs. Sunday Test (per `feedback_prompt_size_policy.md`).
